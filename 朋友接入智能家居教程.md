# AionHome 智能家居接入教程

这份教程给完全没接触过 Home Assistant / Docker 的朋友用。

整体链路是：

```text
小米/其他智能家居设备 -> Home Assistant -> AionHome 的 Home Assistant MCP -> AIon / Connor
```

也就是说，AionHome 不直接登录米家，而是通过 Home Assistant 统一控制智能家居。

## 1. 需要准备什么

需要先准备：

- 一台 Windows 电脑
- AionHome 项目文件
- Docker Desktop
- Python 环境，也就是 AionHome 原本运行需要的环境
- 小米账号，且智能家居设备已经能在米家 App 里看到

Docker Desktop 一般不需要登录账号，拉公开镜像通常可以直接用。

## 2. 安装 Docker Desktop

1. 打开 Docker 官网：https://www.docker.com/products/docker-desktop/
2. 下载 Windows 版 Docker Desktop
3. 按默认选项安装
4. 安装完成后重启电脑
5. 打开 Docker Desktop，等左下角/页面显示 Docker 正常运行

如果提示 WSL 相关组件，按 Docker Desktop 的提示安装即可。

## 3. 启动 Home Assistant

在 AionHome 项目根目录打开 PowerShell。

比如项目路径是：

```text
D:\AionsHome
```

那就在这个目录里右键打开终端，运行：

```powershell
$haConfig = Join-Path (Get-Location) "aion-chat\data\homeassistant-config"
New-Item -ItemType Directory -Force $haConfig
docker run -d --name homeassistant --restart unless-stopped -p 8123:8123 -e TZ=Asia/Shanghai -v "${haConfig}:/config" ghcr.io/home-assistant/home-assistant:stable
```

然后浏览器打开：

```text
http://127.0.0.1:8123
```

第一次进入会让你创建 Home Assistant 本地账号。

这个账号只是 Home Assistant 本地账号，不是米家账号。

## 4. 安装小米官方 Home Assistant 集成

如果你的设备主要在米家里，继续执行下面的命令。

还是在 AionHome 根目录的 PowerShell 里运行：

```powershell
git clone https://github.com/XiaoMi/ha_xiaomi_home.git aion-chat\data\homeassistant-config\ha_xiaomi_home
New-Item -ItemType Directory -Force aion-chat\data\homeassistant-config\custom_components
Copy-Item -Recurse -Force aion-chat\data\homeassistant-config\ha_xiaomi_home\custom_components\xiaomi_home aion-chat\data\homeassistant-config\custom_components\
docker restart homeassistant
```

然后打开 Home Assistant：

```text
http://127.0.0.1:8123
```

进入：

```text
设置 -> 设备与服务 -> 添加集成 -> 搜 Xiaomi Home
```

选择绿色图标的 `Xiaomi Home`，然后用自己的小米账号登录授权。

如果授权时提示 `invalid redirect uri`，或者授权后跳不回来，可以改用这个地址打开 Home Assistant：

```text
http://homeassistant.local:8123
```

如果 `homeassistant.local` 打不开，需要在 Windows hosts 文件里加一行：

```text
127.0.0.1 homeassistant.local
```

hosts 文件位置通常是：

```text
C:\Windows\System32\drivers\etc\hosts
```

修改 hosts 需要管理员权限。

## 5. 创建 Home Assistant 长期访问令牌

进入 Home Assistant 后：

1. 左下角点自己的用户名
2. 拉到最下面
3. 找到 `长期访问令牌` / `Long-Lived Access Tokens`
4. 点击创建令牌
5. 名字可以写 `AionHome`
6. 复制生成出来的 token

注意：这个 token 只显示一次，一定要当场复制保存。

## 6. 把 token 填到 AionHome

新建或修改这个文件：

```text
aion-chat\data\home_assistant_mcp.json
```

内容示例：

```json
{
  "ha_url": "http://127.0.0.1:8123",
  "ha_token": "这里粘贴自己的 Home Assistant 长期令牌",
  "dry_run": false,
  "allowed_domains": [
    "light",
    "switch",
    "fan",
    "cover",
    "climate",
    "humidifier",
    "scene",
    "script"
  ]
}
```

`ha_token` 后面填刚刚复制的长期访问令牌。

这个文件不要上传 GitHub。

## 7. 注册 Home Assistant MCP

新建或修改这个文件：

```text
aion-chat\data\mcp_servers.json
```

内容示例：

```json
{
  "servers": [
    {
      "name": "Home Assistant",
      "type": "stdio",
      "command": "python",
      "args": [
        "D:\\AionsHome\\aion-chat\\mcp_servers\\home_assistant_server.py"
      ],
      "enabled": true,
      "visible": false
    }
  ]
}
```

注意把这里的路径：

```text
D:\AionsHome\aion-chat\mcp_servers\home_assistant_server.py
```

改成你自己电脑上真实的项目路径。

Windows 路径在 JSON 里要写成双反斜杠，比如：

```text
D:\\AionsHome\\aion-chat\\mcp_servers\\home_assistant_server.py
```

## 8. 找到自己家的设备 entity_id

打开 Home Assistant：

```text
开发者工具 -> 状态
```

在搜索框里搜你的设备名字，比如：

```text
客厅灯
卧室灯
空调
```

找到对应的 `entity_id`，它通常长这样：

```text
light.xxx_living_room
switch.xxx_bedroom
climate.xxx_ac
```

把这些 `entity_id` 复制下来。

## 9. 配置 AIon / Connor 可以控制哪些设备

新建或修改：

```text
aion-chat\data\home_assistant_aliases.json
```

示例：

```json
{
  "aliases": {
    "客厅灯": {
      "entity_id": "light.xxx_living_room",
      "notes": "客厅灯"
    },
    "卧室灯": {
      "entity_id": "switch.xxx_bedroom",
      "notes": "卧室灯"
    },
    "空调": {
      "entity_id": "climate.xxx_ac",
      "notes": "空调"
    }
  },
  "groups": {
    "所有灯": [
      "客厅灯",
      "卧室灯"
    ],
    "全部灯": [
      "客厅灯",
      "卧室灯"
    ]
  },
  "blocked_entities": [],
  "blocked_aliases": []
}
```

这里就是智能家居白名单。

只有写在 `aliases` 里的设备，AIon / Connor 才能控制。

`groups` 是组别名。比如用户说“关掉所有灯”，AIon 可以只输出：

```text
[HOME:off|所有灯]
```

后端会自动展开成组里的所有灯逐个关闭。

## 10. 修改 AIon 提示词里的设备列表

有两个地方要改。

第一个：

```text
aion-chat\routes\chat.py
```

第二个：

```text
aion-chat\context_builder.py
```

搜索：

```python
HOME_ALIASES_HINT
```

改成自己家的设备名，比如：

```python
HOME_ALIASES_HINT = (
    "所有灯、客厅灯、卧室灯、空调"
)
```

这里是给 AIon 看的提示词。

真正的控制权限还是以 `home_assistant_aliases.json` 为准。

## 11. 重启 AionHome 并测试

重启 AionHome。

先测试只读状态：

```text
查看客厅灯状态
```

如果成功，再测试开关：

```text
帮我打开客厅灯
帮我关闭客厅灯
帮我关闭所有灯
```

如果配置了空调，可以测试：

```text
帮我把空调开到 26 度
```

## 12. 常见问题

### Docker Desktop 需要登录吗？

一般不需要。

只要 Docker Desktop 正常运行，公开镜像通常可以直接拉取。

### Home Assistant 打不开怎么办？

先试：

```text
http://127.0.0.1:8123
```

再试：

```text
http://homeassistant.local:8123
```

还不行就检查 Docker 容器是否在运行：

```powershell
docker ps
```

如果看不到 `homeassistant`，可以试着重新启动：

```powershell
docker start homeassistant
```

### 小米登录授权跳不回来怎么办？

优先用：

```text
http://homeassistant.local:8123
```

如果打不开 `homeassistant.local`，按第 4 步修改 hosts。

### AIon 说执行了但没反应怎么办？

检查这几个地方：

1. `home_assistant_mcp.json` 里的 token 是否正确
2. `ha_url` 是否是 `http://127.0.0.1:8123`
3. `home_assistant_aliases.json` 里的 `entity_id` 是否复制正确
4. Home Assistant 里手动控制这个设备是否能成功
5. AionHome 是否已经重启

### 可以控制没写进 aliases 的设备吗？

不可以。

这是故意设计的安全限制。

AIon / Connor 只能控制 `home_assistant_aliases.json` 白名单里的设备。

## 13. 不要上传 GitHub 的文件

这些文件包含个人数据、token、设备信息或登录态，不要公开上传：

```text
aion-chat\data\home_assistant_mcp.json
aion-chat\data\home_assistant_aliases.json
aion-chat\data\homeassistant-config\
aion-chat\data\mcp_servers.json
```

如果要打包发给别人，先运行根目录下的：

```text
清理个人数据.bat
```
